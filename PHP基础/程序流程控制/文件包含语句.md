# 文件包含语句

文件包含语句是PHP中一种强大的功能，它允许将一个PHP文件的所有内容（包括代码、HTML、文本等）包含到当前文件的指定位置 。
这对于代码复用（例如将数据库连接、页眉、页脚等公共部分提取出来）非常有用。

# 1 四种文件包含语句
1. `include '文件名';`
2. `require '文件名';`
3. `include_once '文件名';`
4. `require_once '文件名';`
# 2 关键区别
这四种语句可以根据两个维度来区分：**错误处理方式** 和 **是否重复包含**。
## 2.1 错误处理方式 (`include` vs `require`)
这是两者最本质的区别，在于当找不到被包含的文件时，它们的处理方式不同 。
- **`include` 和 `include_once`**： 如果找不到被包含的文件，浏览器会**发出一个警告信息**（Warning），但脚本会**继续执行** 。
    - **适用场景：** 用于包含可选内容，例如模板的某个可选部分。
- **`require` 和 `require_once`**： 如果找不到被包含的文件，浏览器会**发出一个致命错误信息**（Fatal Error），并**中止执行**代码 。
    - **适用场景：** 用于包含必需的核心文件，例如数据库配置文件、核心函数库。如果这些文件丢失，后续代码也无法正常运行。
## 2.2 重复包含 (`_once` vs. 常规)
- **`include` 和 `require`**： 表示**直接把文件包含进来** 。如果在一个脚本中多次调用 `include 'db.php'`，`db.php` 文件的内容将会被包含并执行多次。
- **`include_once` 和 `require_once`**： 表示**只把文件包含进来一次** 。在第一次包含后，PHP会记住这个文件。后续任何对 `include_once` 或 `require_once` 包含同一个文件的尝试都将被忽略。
    - **适用场景：** 这是最常用的方式，特别是包含函数定义或类定义的文件时，可以防止因重复定义而导致的错误。
# 3 路径使用说明
- 上述语句中的 `'文件名'` 可以采用**相对路径**或**绝对路径**两种方式 。
- 需要注意的是，路径分隔符建议采用**斜杠（`/`）** 。
- 若省略路径，则表示仅在当前目录中查找文件 。
