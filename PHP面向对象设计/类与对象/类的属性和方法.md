# 类的属性和方法

# 1 类的属性（成员变量）

属性是类中用于数据存储的变量。
- 声明格式：
```php
var/public/protected/private [static] 变量名;
```
- 访问权限关键字：
    - `var`/ `public`：两者都表示公有属性。`var`是 PHP 4 的旧关键字，现在推荐使用`public`。
    - `protected`：受保护属性。
    - `private`： 私有财产。
- `static`关键词：
    - 用于静态静态属性。`static`前面的只能使用`public`、`protected`或者`private`，不能使用`var` 。
# 2 类的方法（成员函数）
方法是类中用于执行操作的函数。
- 声明格式：
```php
[public/protected/private] [static] function 函数名(形参表)
{
    // 函数体
}
```
访问权限关键字：
- 使用`public`、`protected`或`private`来声明方法的可见性相同。
# 3 访问权限（可见性）
这三个关键字是面向对象封装的核心，它们用于控制类的属性和方法在哪里可以被访问。
1. `public`(公有)
    - 可以在任意位置被访问，包括类的内部、子类中，以及类的外部（即通过对象实例访问）。
    - 如果不写任何关键字，成员默认为`public` 。
2. `protected`（受保护）
    - 只能被自身类（即本类）和派生类（即子类）的成员函数访问。
3. `private`（不）
    - 只能被自身类的成员函数访问。子类和类的外部都无法访问。
# 4 设计实践
一个通用的设计原则：
在进行类设计时，通常将类的属性设置为`private`（私有的），而将方法设置为`public`（公有的） 。
