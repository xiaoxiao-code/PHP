# 方法覆盖

# 1 概念区分：方法覆盖 vs 方法重载

在深入“方法覆盖”之前，教程首先区分了两个容易混淆的概念：
- 重载方法（Overloading）：
    - 指在同一类中，定义了多个名称相同的方法，但它们的形参个数或形参类型有所不同。
    - 教程明确指出：PHP 目前不支持方法的重载 。
- 方法覆盖（Overriding）：
    - 指在父类和子类中，分别定义了相同名称的方法。
# 2 什么是方法覆盖？
方法覆盖是指当一个子类继承一个父类时，子类可以定义一个与父类同名的方法。
- PHP中的定义：教程中指出，在PHP中，只要子类定义的方法与父类中的方法名称相同，无论形参是否相同，都称子类的方法覆盖父类的方法。
- 结果：当子类覆盖了父类的方法时，父类的同名方法在子类中被隐藏起来。当通过子类的对象调用该方法时，执行的是子类中定义的版本。
# 3 方法覆盖的示例
此示例展示了子类`B`如何覆盖父类`A`的属性`$x`和方法`func()`。
代码：
```php
<?php
// 父类
class A 
{
    public $x = "stringA";

    function func($a) // 父类的方法 func(带1个参数)
    {
        echo "父类A";
    }
}

// 子类 B 继承 A
class B extends A
{
    public $x = "stringB"; // 覆盖了父类的属性 $x

    function func() // 覆盖了父类的方法 func(不带参数)
    {
        echo "子类 B";
    }
}

// 创建子类的对象
$b = new B();

// 访问属性 $x
echo $b->x; // 输出: stringB

// 调用方法 func()
$b->func(); // 输出: 子类 B
?>
```
程序说明：
1. 在类中`B`，属性`$x`（值为“stringB”）覆盖了类`A`的属性`$x`（值为“stringA”）。
2. 在类`B`中，方法`func()`（无参数）覆盖了类`A`的方法`func($a)`（有参数）。
3. 因此，当创建`B`类的对象`$b`并调用`$b->x`时`$b->func()`，执行的都是子类`B`中定义的版本。
