# 抽象类

抽象类是面向对象编程中的一个重要概念，它为子类定义了一个必须遵守的“模板”或“规范”。

# 1 抽象方法（抽象方法）
- 定义：抽象方法是指只提供方法的声明，而不提供方法的具体实现（即没有方法体`{}`）。
- 语法格式：
```php
abstract function 函数名(形参表);
```
- - 注意：它以分号`;`结尾，而不是花括号`{}`。
# 2 抽象类（Abstract Class）
- 定义：用`abstract`修饰符声明的类称为抽象类。
- 核心规则：只要一个类中包含一个或多个抽象方法，那么该类必须被声明为抽象类。
# 3 抽象类的特性
Abstract类具有以下三个关键特性：
1. 一个抽象类一般包含一个或抽象方法（也可以包含非多个抽象的、已实现的方法）。
2. 抽象类不能被实例化（即不能使用`new`创建关键字对象）。它只能用来派生子类（即被`extends`）。
3. 当一个子类继承一个抽象类时，该子类必须给出父类中所有抽象方法的具体实现（即必须实现所有没有方法体的方法）。
# 4 抽象类的示例
这里示例定义了一个抽象类`teacher`，并由子类`stu`继承并实现它。
代码：
```php
<?php
// 1. 定义抽象类 teacher
abstract class teacher
{
    // 抽象类可以有已实现的属性
    var $number = "101";
    var $project;

    // 2. 声明抽象方法 (只有声明，没有实现)
    abstract function shownumber();
    abstract function getproject($x);

    // 3. 抽象类也可以有非抽象方法 (已实现)
    function showproject()
    {
        echo $this->project;
    }
}

// 4. 定义子类 stu，继承抽象类 teacher
class stu extends teacher
{
    // 5. 子类 *必须* 实现父类的所有抽象方法

    // 实现 shownumber()
    function shownumber()
    {
        echo $this->number;
    }

    // 实现 getproject()
    function getproject($x)
    {
        $this->project = $x;
    }
}

// 6. 抽象类 teacher 不能被实例化
// $t = new teacher; // (这行代码会产生致命错误)

// 7. 子类 stu 已经被实例化
$obj = new stu;
$obj->shownumber(); // 输出 101
$obj->getproject("计算机");
$obj->showproject(); // 输出 计算机
?>
```
