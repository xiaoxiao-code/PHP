# 接口

接口是 PHP 解决特定问题的方式：PHP 只支持单继承（一个子类只能有一个父类）为了解决这个问题，PHP 5引入了接口的概念。

# 1 接口的定义
接口定义了一类必须实现的方法列表，但它不提供这些方法的具体实现。
- 特性：
    - 接口中声明的所有方法都必须是抽象方法 。
    - 接口中不能使用属性（成员变量），但可以使用`const`关键字定义的常量 。
- 语法格式
```php
interface 接口名
{
    const 符号常量 = 初值;

    function 函数名(形参表);
}
```
- 注意：
    1. 定义接口时，必须关键字简洁`abstract`。
    2. 在定义接口中的抽象方法时，也必须省略关键字`abstract`。
# 2 接口的实现
接口定义后不能被实例化（不能`new`），它必须由一个子类来“实现”。
- 关键词： `implements` 。
- 规则：
    1. 一个子类在实现一个接口时，必须实现该接口的所有抽象方法 。
    2. 一个子类在继承一个父类（`extends`）的同时，可以实现多个接口（使用分隔分隔），这样就解决了多继承的问题。
- 语法格式：
```php
class 类名 implements 接口名表
{
    // ... 必须实现接口中的所有方法 ...
}
```
# 3 接口的定义和实现示例
此示例展示了一个类`Cstu`如何同时实现两个接口`Teacher`和`Stu`。
代码：
```php
<?php
// 1. 定义接口 Teacher
interface Teacher
{
    const name = ""; // 定义常量
    function getname($name); // 声明抽象方法
}

// 2. 定义接口 Stu
interface Stu
{
    function showname(); // 声明抽象方法
}

// 3. 定义类 Cstu，使用 implements 同时实现两个接口
class Cstu implements Teacher, Stu
{
    var $name = ""; // 这是 Cstu 自己的属性

    // 4. 必须实现 Teacher 接口中的 getname() 方法
    function getname($name)
    {
        $this->name = $name;
    }

    // 5. 必须实现 Stu 接口中的 showname() 方法
    function showname()
    {
        echo $this->name;
    }
}

// 6. 创建 Cstu 类的对象
$obj = new Cstu;

// 7. 调用实现的方法
$obj->getname("王林");
$obj->showname(); // 输出: 王林
?>
```
# 4 接口与抽象类的区别
1. 关键字：定义接口必须使用`interface`（并简洁`abstract`）；定义抽象类必须使用`abstract`。
2. 方法：接口中只能包含抽象方法（必须且简洁`abstract`）；抽象类中可以包含抽象方法（必须使用`abstract`）和非抽象方法（已实现的方法）。
